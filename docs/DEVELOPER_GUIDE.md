# Yapakit Developer Guide

## Project Overview

Yapakit is a multi-tenant SaaS Point of Sale (POS) and Restaurant Operating System.

### Core Architecture

- **Backend:** Node.js, Express, TypeScript, MongoDB (Mongoose).
- **Frontend:** React (Vite), TypeScript, TanStack Query v5, Zustand, Tailwind CSS, Recharts.
- **Multitenancy:** Isolated by `restaurantId` (Tenant ID). The `x-restaurant-id` header is required for all tenant-scoped API requests.
- **Security:** JWT-based. Supports **Role Impersonation** for global roles (Superadmin, Support, Sales). Cross-Site Scripting (XSS) protection with `helmet`.

## Getting Started

### Installation

```bash
# Backend
cd server && npm install

# Frontend
cd client && npm install
```

### Environment Setup

Create a `.env` file in the `server` directory:

```env
PORT=5005
MONGO_URI=mongodb://localhost:27017/yapakit
JWT_SECRET=your_secret_key
```

### Development Commands

```bash
# Run seeder (Wipes DB and creates demo data)
npm run seed

# Start Backend Dev Server
npm run dev

# Start Frontend Dev Server
cd client && npm run dev
```

## 🚀 Specialized Modules

### 🎫 Support Ticket System
Global admins can manage support requests from restaurants through the **Support** portal. Support agents can resolve technical issues while Superadmins oversee the entire system.

### 💳 Subscription & Sales
Yapakit includes a built-in subscription engine with plans (Basic, Pro, Plus, Custom). New restaurants start with a 14-day trial automatically. Sales agents can manage these plans through the **Sales** portal.

### 📊 Advanced Analytics
Built using **Recharts**, the Manager Dashboard provides real-time insights into sales, order volume, and inventory levels.

> [!NOTE]
> For **Impersonation Mode**, use the `x-restaurant-id` header to specify which restaurant you want to manage. The system will bypass membership checks for `superadmin`, `support`, and `sales`.
